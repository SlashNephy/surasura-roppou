# データのエクスポート・インポート

すらすら六法は、端末に保存した7分類のデータを一つのJSONファイルにまとめ、別の端末へ移動できます。

## 操作方法

設定の「エクスポート / インポート」を開きます。

エクスポートでは、「JSONをエクスポート」を選ぶと現在の保存データをファイルに書き出します。
保存リストにある同名のボタンは、このexportを直接実行するショートカットです。

インポートでは、version 2のJSONファイルを選択します。
ファイルのversion、出力日時、7分類ごとの件数がpreviewに表示されるため、内容を確認してから「この内容をインポート」を選びます。
ファイルを選択しただけでは保存データを変更しません。

## 移動するデータ

一つのJSONファイルに次の7分類を含めます。

- 保存法令本文
- ブックマーク
- コレクション
- メモ
- 学習カード
- 回答ログ
- 学習セッション

CardScheduleはファイルに含めません。
CardScheduleは、インポート後にReviewLogから再構築します。

## プライバシー

JSONの作成、検証、インポートは端末内で処理し、サーバーへ送信しません。

ただし、書き出したファイル自体には利用者が保存した法令本文、メモ、学習履歴などが入ります。
ファイルの保管場所と共有先に注意してください。

## 対応versionとschema

現在対応する形式はversion 2だけです。
version 1は利用者向けの配布実績がないため、変換も受理もしません。

version 2のcanonical schemaは、[saved-data-export-v2.schema.json](./schemas/saved-data-export-v2.schema.json)です。
連携実装は、このschemaに対してファイル全体を検証してからインポートを実行します。

任意フィールドの追加を含め、JSON構造を変更するときはversionを上げます。
新しいversionへの対応は明示的に実装し、未知のversionを推測して処理しません。

## mergeと失敗時の動作

インポートは、同じIDのデータをファイル側の内容で上書きします。
ファイルに含まれない既存データは、そのまま保持します。

インポート前にファイル全体のversion、schema、ID、参照関係を検証します。
検証を通過したデータは一つのtransactionで書き込みます。
途中で失敗した場合はtransaction全体を取り消し、保存データを一件も変更しません。
