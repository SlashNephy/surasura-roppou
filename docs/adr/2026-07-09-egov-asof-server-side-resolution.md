# e-Gov API v2 の asof をサーバー側の版解決として利用する

Date: 2026-07-09

## Status

Accepted

## Context

Issue #70 では、学習年度の基準日に基づき「施行日 ≤ 基準日 の最新版」で法令を表示する。
[改正・基準日の参照設計](../superpowers/specs/2026-07-07-revision-and-asof-design.md) の 8 章は、e-Gov API v2 が版指定取得を提供しない場合に、取得のたびに前回スナップショットと指紋比較する縮退方式を採ると定めていた。実装着手時に API 仕様を検証する必要があった。

## Decision

2026-07-09 に本番 API（民法 `129AC0000000089`）へ実リクエストして次を確認し、`asof` によるサーバー側版解決を採用する。

- `GET /law_data/{lawId}?asof=YYYY-MM-DD` は「施行日 ≤ asof の最新版」を返す（`asof=2020-06-01` → 2020-04-01 施行版）。
- `asof` 未指定は現行施行版を返す（今日の asof と同結果）。未来日は未施行の最新予定版を返す。
- `asof` の下限は **2017-04-01**。それ未満は HTTP 400（`code=400044`）。

したがって参照設計 8 章の縮退方式（指紋差分）は採らない。クライアントは基準日を `asof` として渡すだけでよく、施行日比較は実装しない。

## Consequences

- 基準日の版解決は `EgovLawRepository.getLaw(lawId, { asOf })` への配線に集約され、クライアントに版比較ロジックを持たない。
- 基準日入力は 2017-04-01 を下限として検証する（`src/core/settings/base-date.ts` の `earliestBaseDate`）。下限未満や不正日は保存しない。
- オフライン保存法令は法令ごとに 1 版のみ保持するため、オンライン解決版と食い違い得る。この再同期（「更新あり」）は #71/#20 の責務とし、本 issue では表示上のラベリングに留める。
- 条文指紋による改変検知は保存物（ブックマーク・メモ・カード）向けの別機能として #71/#20 で扱う。
