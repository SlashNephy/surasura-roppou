# 法令参照 Resolver 仕様

Status: Approved (設計検討セッション 2026-07-07)
Last updated: 2026-07-07

関連ドキュメント:

- [Design Doc](../../design-doc.md) の 7.6 章と 11 章のドラフトを確定させるものである。
- UI 側の前提: [2026-07-06-ui-wireframe-design.md](2026-07-06-ui-wireframe-design.md) 5 章（確信度が高くても自動遷移せず、Enter のデフォルトフォーカスで表現する）。

## 1. 決定事項の要約

- **対応範囲**: design-doc 7.6 の MVP 表記のみとする。文脈依存参照（前条、同法）、ローマ数字（憲21Ⅰ）、条番号部の漢数字（一条二項）は対象外。
- **略称辞書**: アプリ同梱の固定辞書（50〜100 件、行政書士プリセット優先）を第一とし、辞書で解決できない法令名はオンライン時に限り e-Gov の法令名検索 API で部分一致候補を補う。
- **確信度**: 数値スコアではなく **exact / prefix / fuzzy の離散 3 ランク**で表現する。
- **仕様の本体はテストフィクスチャ**とし、約 100 件の「入力 → 期待候補列」の表を受け入れ基準にする。

## 2. パイプライン

```text
入力文字列
  → 正規化（NFKC → 空白除去 → 全角数字を半角化）
  → 法令名部の最長一致（辞書照合）
  → 条番号部の構文解析
  → 候補生成とランク付け
  → （辞書ミス時・オンライン時のみ）e-Gov 法令名検索でフォールバック候補
```

正規化は既存の readability 変換器の部品を再利用する。
漢数字から算用数字への変換は条番号部では行わない（対象外の明示。誤変換で別条文を指すリスクを避ける）。

## 3. 文法

正規化後の入力に対して、次の 2 部構成のみを受理する。

```text
参照     := 法令名部 条番号部?
条番号部 := 条 項? 号?
条       := "第"? 数字列 "条"? 枝番*        // 709 / 709条 / 第709条 / 242条の2
枝番     := "の" 数字列
項       := 数字列 "項"
号       := 数字列 "号"
```

- 「条」の省略（`民709`）は、法令名部の直後が数字列で始まる場合に許す。
- 項・号は条が確定した後にのみ受理する（`民2項` のような入力は非参照として扱う）。
- 条番号部を持たない入力（`国家賠償法`）は参照ではなく法令そのものへの候補として扱う（article なしの候補を返す）。

## 4. 略称辞書

```ts
interface LawAliasEntry {
  alias: string; // 正式名もエントリとして持つ
  lawId: string;
  lawTitle: string;
  kind: "official" | "abbreviation";
}
```

- 初期辞書は design-doc 11.2 の表を核に、行政書士試験の頻出法令（憲法、民法、行政法パック、商法/会社法、基礎法学まわり）を優先して 50〜100 件を同梱する。
- 同一 alias が複数法令を指す場合は、両方を候補として返す（辞書側で優先順位を持たない）。
- 辞書は TypeScript の定数として同梱し、更新はアプリのリリースで行う（BFF 配信は [ADR 2026-07-07](../../adr/2026-07-07-defer-bff-introduction.md) により先送り）。

## 5. ランクと並び順

照合は 2 方向で行い、結果を 3 ランクに割り当てる。

| ランク | 条件                                                               |
| ------ | ------------------------------------------------------------------ |
| exact  | 入力の法令名部が辞書エントリ（正式名または略称）に完全一致         |
| prefix | 入力の法令名部が辞書エントリの前方一致（`国家賠償` → 国家賠償法）  |
| fuzzy  | 辞書エントリへの部分一致、または e-Gov 法令名検索 API の返した候補 |

- 同ランク内の並び: 最近開いた法令 → オフライン保存済みの法令 → 法令名の昇順。
- UI との接続: 先頭候補が exact のとき Enter のデフォルトフォーカスにする。fuzzy しかないときは「確信度 低」を表示する。候補ゼロのときは「法令名として検索」への導線を出し、ゼロヒット画面にしない。

## 6. 出力型

design-doc 10 章のドラフト型を次のとおり改訂する（`score: number` と `confidence: number` を廃止）。

```ts
type CandidateRank = "exact" | "prefix" | "fuzzy";

interface LawReferenceCandidate {
  lawId: string;
  lawTitle: string;
  article?: string; // "709" | "242-2"（枝番はハイフン区切りの内部表現）
  paragraph?: string;
  item?: string;
  rank: CandidateRank;
  reason: string[]; // 例: ["略称『国賠』に一致"]
}
```

article の文字列表現（枝番はハイフン区切り）は、既存 `src/core/domain/references.ts` の `buildLawArticleUrl` が URL の `:article` に使う表現と同一とする。
Resolver の出力・URL・保存物のアンカーで article 表現を揃え、変換を挟まない。

既存実装からの移行: 現行 `src/core/domain/models.ts` の `LawReferenceCandidate` は `score: number` を、`DetectedLawReference` は `confidence: number` を持つ。
M4 の実装で `score` を `rank` に置き換え、`confidence` を削除する（`LawReferenceCandidate extends Partial<ArticleReference>` の構造は維持する）。
利用箇所は未実装のため、この置き換えに実行時の互換問題はない。テストフィクスチャの移行は 7 章で扱う。

将来 OCR（M5）で数値的な確信度が必要になった場合は、rank を保ったまま同ランク内の tiebreaker として数値を追加する（rank の 3 値は UI 契約として維持する）。

## 7. テストフィクスチャ計画

`src/test/fixtures/lawReferences.ts` を拡充し、次のカテゴリで約 100 件を定義する。
期待値は「先頭 3 候補の (lawId, article, rank)」の列とする。

| カテゴリ                             | 件数目安 | 例                                |
| ------------------------------------ | -------- | --------------------------------- |
| 正式名 + 条                          | 10       | 国家賠償法1条                     |
| 略称 + 条（辞書内）                  | 20       | 国賠法1条、行手法14条             |
| 条の省略形                           | 10       | 民709、国賠1                      |
| 項・号付き                           | 15       | 憲法21条1項、行政手続法14条1項1号 |
| 枝番                                 | 10       | 地方自治法242条の2                |
| 辞書外の法令名（API フォールバック） | 10       | 個人情報の保護に関する法律20条    |
| 曖昧（同 alias 複数、prefix 競合）   | 10       | 商8条                             |
| 非参照入力                           | 10       | ただの単語、数字のみ、空文字      |
| 正規化系                             | 5        | 全角数字、空白混じり              |

受け入れ基準はフィクスチャ全件の一致とする。
e-Gov フォールバックはフィクスチャではモック応答を使い、オフライン時はフォールバックが発火しないことも 1 件含める。

既存フィクスチャからの移行: 現行の `lawReferences.ts` は `expected.confidenceFloor` を持つ単一パース結果の schema であり、本仕様の「先頭 3 候補 + rank」schema に**置き換える**（confidenceFloor は廃止）。
現行フィクスチャに含まれる条番号部の漢数字ケース（`民法第七百九条の二` など）は本仕様では対象外のため、`deferred`（M5 の OCR 正規化で再検討）として別グループに移し、MVP の受け入れ対象から外す。

## 8. 対象外（明示）

- 文脈依存参照: 前条、次条、前項、同法、本文、ただし書。
- ローマ数字表記: 憲21Ⅰ。
- 条番号部の漢数字: 一条二項三号、第十二条の二。
- 範囲参照: 1条から3条まで。
- 別表・様式への参照: 別表第一。

これらはパーサーが受理しなかった入力として法令名検索へ落ちる（エラーにはしない）。
