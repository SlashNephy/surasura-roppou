# 見やすいモードの構造見出しと目次の数字変換設計

Status: Approved（2026-07-15 の設計確認）
Last updated: 2026-07-15

関連 Issue: [#95 「見やすいモード」で、章や節、TOC の漢数字も置き換える](https://github.com/SlashNephy/surasura-roppou/issues/95)

## 1. 背景

現行の「読みやすい表示」は、条、項、号、別表、別記様式、日付、法令番号に含まれる漢数字を表示レイヤーで算用数字へ変換する。

しかし、編、章、節、款、目の構造見出しは変換対象に含まれないため、本文では「第一章」、目次では「第一章」や「第一条」が原文のまま表示される。

本文と目次で表示規則が異なる状態を解消し、「読みやすい表示」の選択を法令の読書領域全体へ反映する。

## 2. 表示要件

「読みやすい表示」では、次の構造見出し番号を算用数字へ変換する。

| 原文     | 読みやすい表示 |
| -------- | -------------- |
| `第一編` | `第1編`        |
| `第二章` | `第2章`        |
| `第三節` | `第3節`        |
| `第四款` | `第4款`        |
| `第五目` | `第5目`        |

既存の条、項、号、別表、別記様式、日付、法令番号の変換規則は維持する。

「原文表示」では、本文と目次の文字列を変換しない。

変換は本文見出し、デスクトップ目次、モバイル目次へ同じ規則で適用する。

## 3. 変換境界

原文の `LawNode` と目次項目は変更せず、描画時だけ可読性変換を適用する。

この境界により、e-Gov から取得した原文、保存済み法令、条文 URL の識別値を保持したまま表示だけを切り替えられる。

構造見出しの数字変換は、既存の共通可読性変換へ追加する。

目次コンポーネントは表示モードを受け取り、各目次項目のタイトルを描画するときに本文と同じ表示変換を呼び出す。

デスクトップ目次とモバイル目次は同じ目次コンポーネントを利用しているため、両方へ同じ表示モードを渡す。

## 4. 対象コンポーネント

### 4.1 共通可読性変換

共通可読性変換は、`第`、漢数字、構造単位の順に並ぶ表記を認識する。

構造単位は編、章、節、款、目に限定し、一般の文章中に現れる漢数字は変換しない。

不正な漢数字は既存の数値変換規則に従って原文のまま保持する。

### 4.2 法令本文

法令本文は既に表示モードを見出しへ適用しているため、共通可読性変換の拡張によって構造見出しも変換される。

見出しの階層、書体、余白、アクセシビリティ上の見出しレベルは変更しない。

### 4.3 法令目次

法令目次は `displayMode` を受け取り、表示タイトルだけを変換する。

条文選択に使う `articleNumber`、選択中表示に使う比較値、目次の階層構造は変更しない。

表示モードを切り替えたときは React の再描画によって本文と目次が同時に更新される。

## 5. エラー処理

今回の変更は新しい入出力や非同期処理を追加しないため、新しいエラー表示は設けない。

数値として解釈できない表記は変換に失敗させず、既存どおり元の文字列を返す。

## 6. テスト

共通可読性変換の table-driven test で、編、章、節、款、目の代表例と不正な漢数字の保持を検証する。

法令本文の Testing Library テストで、「読みやすい表示」の構造見出しが算用数字になり、「原文表示」では漢数字のままになることを検証する。

法令目次の Testing Library テストで、入れ子の構造見出しと条タイトルが表示モードに連動することを検証する。

法令ビューワー画面の結合テストで、表示モード切替後に本文と目次が同時に切り替わることを検証する。

実装後は `playwright-cli open --headed` で実画面を開き、デスクトップ目次、モバイル目次、本文見出し、原文表示への復帰を操作して確認する。

## 7. スコープ外

項や号の単独マーカー表記は今回変更しない。

条文参照の解析、法令内ジャンプ、検索インデックス、保存データの形式は変更しない。

目次や本文のレイアウト、色、書体、操作導線は変更しない。

## 8. 受け入れ条件

1. 「読みやすい表示」で編、章、節、款、目の番号が本文と目次の両方で算用数字になる。
2. 「読みやすい表示」で目次の条タイトルも既存の条番号変換に従う。
3. 「原文表示」で本文と目次が原文の漢数字へ戻る。
4. デスクトップ目次とモバイル目次が同じ表示結果になる。
5. 条文選択、URL 遷移、目次階層に既存動作からの変更がない。
