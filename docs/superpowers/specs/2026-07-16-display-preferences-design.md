## 1. 目的

現行の設定画面は、文字サイズ、行間、テーマの値を表示するだけで変更操作を持たない。

Issue #112 では、三種類の選択肢を設定画面で変更できるようにし、利用者が選んだ値を端末へ保存する。

設定は法令本文だけに限定せず、目次、検索結果、ダイアログ、設定画面を含むアプリ全体の表示レイヤーへ適用する。

## 2. 完了境界

次の項目を実装対象とする。

- 文字サイズの「標準」「やや大きい」「大きい」
- 行間の「標準」「ゆったり」「広い」
- テーマの「システム」「ライト」「ダーク」
- 名前空間付き `localStorage` への保存と次回起動時の復元
- 「システム」選択時の OS テーマ変更への追従
- 法令本文、目次、検索結果、ダイアログを含む主要表示面への反映
- 320 CSS px、200% 拡大、desktop、mobile の実画面確認

次の項目は実装対象に含めない。

- 法令原文、正規化済みノード、保存データの変更
- コピー内容、export、import の表示設定依存化
- 表示設定のサーバー同期または analytics 送信
- PWA の `theme-color` とアイコン背景色の変更
- 既定の「読みやすい表示」「原文表示」の永続化

PWA の色は Issue #94、表示モードの永続化は別 Issue の責務である。

## 3. 選択肢と既定値

表示設定は次の値を持つ。

| 項目       | 内部値        | 表示名     | CSS 上の値            |
| ---------- | ------------- | ---------- | --------------------- |
| 文字サイズ | `standard`    | 標準       | `100%`（16px 相当）   |
| 文字サイズ | `large`       | やや大きい | `112.5%`（18px 相当） |
| 文字サイズ | `extra-large` | 大きい     | `125%`（20px 相当）   |
| 行間       | `standard`    | 標準       | `1.5`                 |
| 行間       | `relaxed`     | ゆったり   | `1.75`                |
| 行間       | `wide`        | 広い       | `2`                   |
| テーマ     | `system`      | システム   | OS の配色へ追従       |
| テーマ     | `light`       | ライト     | ライトテーマへ固定    |
| テーマ     | `dark`        | ダーク     | ダークテーマへ固定    |

初回起動時は、文字サイズと行間に `standard`、テーマに `system` を使う。

「システム」は初回起動時だけの自動判定ではない。
利用者が「システム」を明示的に選び直した場合も、OS の配色変更へ追従する。

## 4. 保存契約

表示設定は、項目ごとに独立した名前空間付きキーへ保存する。

| 項目       | キー                            |
| ---------- | ------------------------------- |
| 文字サイズ | `surasura:display:font-size`    |
| 行間       | `surasura:display:line-spacing` |
| テーマ     | `surasura:display:theme`        |

初回起動時はキーを作らず、読み取り時に既定値を補う。
利用者が変更した項目だけを保存するため、文字サイズを変更しても未選択のテーマ値は書き込まない。

保存値は既知の union に含まれる文字列だけを有効とする。
不正な値または未定義の値を読んだ場合は、その項目だけを既定値へ戻し、他の項目は維持する。

同一タブの変更は購読者へ直ちに通知する。
別タブの変更は `storage` イベントで受け取り、開いている画面へ反映する。

テーマの保存、OS 配色の監視、`dark` class の付与には、既存依存の `next-themes` を使う。
`ThemeProvider` の `storageKey` には `surasura:display:theme`、`defaultTheme` には `system` を指定する。

## 5. React への接続

アプリルートへ **表示設定 Provider** を追加する。
Provider は文字サイズと行間の外部ストアを購読し、テーマ Provider と Router を同じ境界の内側へ置く。

文字サイズと行間は、`document.documentElement` の次の属性へ反映する。

- `data-font-size="standard | large | extra-large"`
- `data-line-spacing="standard | relaxed | wide"`

属性は layout effect で更新し、React の描画から最初のブラウザー描画までに適用する。
テーマは `next-themes` が `document.documentElement` の `dark` class を管理する。

設定の読み取りと更新は app 専用 hook から行う。
設定画面が `localStorage` や DOM 属性を直接操作しない構造にする。

## 6. CSS の反映範囲

文字サイズは `html` の `font-size` を切り替え、`rem` を使う既存の文字、余白、コントロールを同じ比率で拡大する。

固定 px の文字指定は rem 相当へ変更する。
これにより、設定値から取り残される小さいラベルを作らない。

行間は CSS custom property に保持し、本文や説明文など、複数行になり得る文字へ適用する。
既存の `leading-5` から `leading-8` のような固定行間は、主要表示面では表示設定を参照する相対指定へ置き換える。

一行見出し、アイコン、コントロール内部まで一律に広げると、配置とタッチ領域の関係が崩れる。
そのため、見出しの `leading-none` など、複数行の読みやすさを目的としない指定は維持する。

次の表示面では、文字サイズと行間の両方が変化することを確認する。

- 法令本文と法令メタデータ
- 法令目次
- 検索ページの候補と説明
- 検索ダイアログの候補と説明
- 設定画面
- 共通ダイアログ内の本文と説明

テーマは既存のライトとダークの color token を使う。
コンポーネント固有の色を追加せず、現行の `dark` variant を有効にする。

## 7. 設定画面

設定画面の「表示」グループは、文字サイズ、行間、テーマの native select と、既存の「既定の表示」行で構成する。

各 select は可視ラベルと関連付ける。
native select を使うことで、Tab、Shift+Tab、矢印キー、Enter、ポインター、タッチの標準操作を維持する。

desktop では select を内容に応じた幅にし、mobile では利用可能な横幅へ広げる。
ラベル、select、説明を縦方向へ並べ、320 CSS px で二方向スクロールを発生させない。

選択は保存ボタンを介さず、変更時に直ちに保存して全画面へ反映する。
設定画面自体も同じ Provider の内側に置くため、変更後の文字サイズと行間で再配置される。

テーマの説明は選択値に合わせて切り替える。

- システム：端末の外観設定に合わせる。
- ライト：端末の外観設定にかかわらずライトで表示する。
- ダーク：端末の外観設定にかかわらずダークで表示する。

説明によって、「システム」だけが OS 変更へ追従する契約を画面上で理解できるようにする。

## 8. データ境界

表示設定は表示レイヤーだけに作用する。

法令ドメインモデル、e-Gov レスポンス、正規化済みノード、IndexedDB の保存値には表示設定を渡さない。
コピーと export は保持済みの原文または既存の整形結果を使い、DOM の見た目から文字列を作らない。

表示設定を URL、ログ、例外メッセージ、analytics payload に含めない。
本 Issue では analytics SDK、送信 endpoint、イベントを追加しない。

## 9. テスト方針

設定ストアは table testing に近い形で次を検証する。

- 保存値がない場合の既定値
- 三種類の正常値の保存と復元
- 不正値を項目単位で既定値へ戻す処理
- 同一タブの購読通知
- 別タブの `storage` イベント

Provider のテストでは、設定値と `document.documentElement` の属性を公開 DOM の状態として検証する。
テーマは `matchMedia` を切り替え、「システム」だけが OS 配色へ追従し、固定テーマは変化しないことを検証する。

設定画面の Testing Library テストでは、実際に select を変更して次を確認する。

- 選択値の可視表示
- `localStorage` への保存
- DOM 属性とテーマ class の即時変更
- Router の再マウント後の復元
- 選択値に対応するテーマ説明

主要表示面の統合テストでは、法令本文、目次、検索結果、検索ダイアログを実際に表示する。
表示設定を変更しても、表示中の原文、保存値、コピー結果が変わらないことを公開操作から検証する。

ソースコードや設定ファイルの文字列を探索するだけのテストは追加しない。

## 10. 実画面検証

実装後は、システムにインストールされた `playwright-cli` を `open --headed` で起動し、確立したセッションを再利用する。

次の操作をキーボードとポインターの両方で確認する。

1. 三種類の文字サイズを順に選択する。
2. 三種類の行間を順に選択する。
3. 「システム」で OS テーマ変更へ追従することを確認する。
4. 「ライト」と「ダーク」で OS テーマ変更へ追従しないことを確認する。
5. reload 後に選択値が復元されることを確認する。
6. 法令本文、目次、検索結果、検索ダイアログを開いて同じ設定を確認する。
7. コピー操作を実行し、表示設定によって内容が変わらないことを確認する。

desktop、mobile、320 CSS px、200% 拡大で、情報または操作が失われず、ページ全体に二方向スクロールが発生しないことを確認する。

法令ビューアの確認には、e-Gov API から取得した実レスポンスを使う。
desktop と mobile のスクリーンショットを保存し、`gh image upload` で PR 本文へ添付する。

## 11. 完了前の確認

コミット直前にプロジェクト標準の品質チェックを実行する。

```bash
pnpm run typecheck
pnpm run lint
pnpm run format:check
pnpm test
git diff --check
```

PR 作成前に Antigravity review を一回実行する。

```bash
pnpm run review:antigravity
```

Antigravity review を踏まえて修正した場合は、通常の品質チェックを再実行し、Antigravity review は再実行しない。
