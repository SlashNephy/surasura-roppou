# 出典明記の共通フッターの設計

Status: Approved (設計検討セッション 2026-07-09)
Last updated: 2026-07-09

関連ドキュメント:

- 対応 Issue: [#77 フッターに出典の明記](https://github.com/SlashNephy/surasura-roppou/issues/77)。
- 根拠: [e-Gov 法令検索 利用規約 1.1 出典の記載について](https://laws.e-gov.go.jp/terms/)。コンテンツ利用時に出典の記載を求めている。
- データ取得元: e-Gov 法令検索 API v2（`https://laws.e-gov.go.jp/api/2`、`src/core/egov/repository.ts`）。

## 1. 決定事項の要約

- 全ルート共通レイアウトである `AppShell` に、`contentinfo` ランドマークとなる共通フッターを 1 つ追加する。どの画面を切り取っても出典が表示される状態にし、e-Gov 利用規約に確実に沿う。
- フッターの内容は **出典表記 + 免責文**の 2 行とする。出典は e-Gov 規約の書式 `出典：{名称}（{URL}）` に準拠し、`https://laws.e-gov.go.jp` へのリンクにする。
- 現状バラバラな個別の出典表記（法令ビューアの独自フッター、設定画面の段落）は**共通フッターへ一本化**し、重複を削除する。
- 法令ビューアの独自フッターが持っていた「取得日」は文書ヘッダ（`LawDocumentView`）に残るため失われない。「読みやすい表示の加工注記」のみ削除する（今回は移設せず、必要なら別 Issue とする）。

## 4. フッターの内容と体裁

表示内容は次の 2 行とする。

```
出典：e-Gov 法令検索（https://laws.e-gov.go.jp）
本アプリは学習補助であり、法的助言を提供するものではありません
```

- 出典行は `出典：` に続けて `e-Gov 法令検索（https://laws.e-gov.go.jp）` 全体を `<a>` リンクにする。リンク先は `https://laws.e-gov.go.jp`、`target="_blank"` かつ `rel="noreferrer"`。URL を可視テキストとして表示することで、e-Gov 規約の「URL を記載する」要件を満たす。
- 免責行は設定画面から移設する既存の文言 `本アプリは学習補助であり、法的助言を提供するものではありません` を用いる。
- スタイルは既存ビューアフッターを踏襲し `border-t bg-popover text-xs text-muted-foreground`、内側を `max-w-4xl` で中央寄せする。
- 実装は `Header` / `MobileNavigation` と同様に `AppShell.tsx` 内のインライン `Footer` コンポーネントとし、既存パターンに合わせる。

## 5. レイアウト（モバイルナビとの重なり対策）

`AppShell` のルート構造を、スティッキーフッターになるよう flex-column に変更する。

```
<div class="flex min-h-dvh flex-col ...">   ← flex-col でフッターを最下部へ
  <Header />
  <main class="min-w-0 flex-1">...</main>    ← flex-1 で高さを占有（既存の pb-20 は撤去）
  <Footer />                                  ← モバイルでは固定ナビ分の下部余白を確保
  <MobileNavigation />                        ← fixed bottom-0（md 未満のみ表示）
  <PwaUpdatePrompt />
</div>
```

- 短いページでもフッターが画面最下部に張り付く。
- モバイルの `fixed bottom-0` ナビにフッター文言が隠れないよう、現状 `main` が持っていた下部余白（`pb-20` 相当 + `env(safe-area-inset-bottom)`）を `Footer` 側へ移す。`md` 以上ではモバイルナビが消えるため通常余白に戻す。

## 8. アクセシビリティ上の補足

`<footer>` は `article` / `section` / `main` / `aside` / `nav` の子孫でないときにのみ `contentinfo` ランドマークになる。撤去する法令ビューアのフッターはページ本文内にネストしていたためランドマークではなかった。`AppShell` 直下に置く新フッターは正式な `contentinfo` ランドマークとなり、スクリーンリーダー利用者がページ情報へ直接移動できるようになる。
