# e-Gov データ取得層 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** e-Gov 法令 API Version 2 の JSON API を使い、UI が `LawRepository` 経由で法令一覧・本文・メタデータを取得できるようにする。

**Architecture:** `src/core/egov` に API レスポンスの取得、検証、ドメインモデルへの正規化を閉じ込める。外部には `LawRepository` interface と `createEgovLawRepository` を公開し、UI は e-Gov 固有のレスポンス形に依存しない。

**Tech Stack:** TypeScript 6, Fetch API, Vitest, e-Gov 法令 API Version 2 JSON response.
