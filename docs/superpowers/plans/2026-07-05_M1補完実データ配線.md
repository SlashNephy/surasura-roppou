# M1 補完 実データ配線 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** M0/M1 完了判定の漏れを補い、法令ビューアが `LawRepository` 経由で実 e-Gov データを取得して表示できる状態にする。

**Architecture:** `core/egov` の repository はそのまま利用し、`app` 側に小さな viewer loader 境界を追加する。`LawViewerPage` は route params と async state 管理だけを担当し、`LawViewerPageContent` は既存どおり表示専用に保つ。canonical URL は現行 route に合わせて e-Gov `lawId` を採用し、設計文書と task index を同期する。

**Tech Stack:** React 19, TanStack Router, TypeScript 6, Vitest, Testing Library, e-Gov 法令 API Version 2, Fetch API.
