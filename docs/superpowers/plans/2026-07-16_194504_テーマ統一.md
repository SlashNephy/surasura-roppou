# Issue #94 テーマ統一 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** PWA アイコン、スマートフォンのシステム UI、起動背景を、すらすら六法の既存ブランド配色へ統一する。

**Architecture:** 実行時のテーマ設定とは分離し、読み込み前に参照される HTML、Web App Manifest、SVG の静的契約を同じ色へ更新する。新しい生成処理は追加せず、設定オブジェクトのテスト、production build の生成物、実HTTP応答、headedブラウザで境界間の一致を検証する。

**Tech Stack:** TypeScript 6、Vite 8、vite-plugin-pwa、Vitest、SVG、playwright-cli、GitHub CLI

---

## File Structure

- `src/core/pwa/config.test.ts`: 生成 manifest が公開するブランド色と起動背景の契約を検証する。
- `src/core/pwa/config.ts`: vite-plugin-pwa が生成する manifest の正本を保持する。
- `public/manifest.webmanifest`: 静的配信時の manifest 契約を生成設定と一致させる。
- `public/pwa.svg`: favicon と maskable PWA アイコンの背景色を保持する。
- `index.html`: 初回描画前からモバイルブラウザが読む固定 `theme-color` を保持する。
