# 表示フォント設定 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 法令本文の「本文フォント」とアプリ全体の「UI フォント」を設定画面から変更できるようにする（Issue #134）。

**Architecture:** 既存の表示設定基盤（localStorage + `useSyncExternalStore` + `data-*` 属性）に 2 項目を追加する。フォントは Fontsource パッケージで自己ホストし、CSS は既存の色トークンと同じ二段変数パターン（`@theme inline` が `:root` の変数を参照）で実行時に切り替える。オフラインは Workbox の runtimeCaching（CacheFirst）で扱う。

**Tech Stack:** React 19 / Vite 8 / TypeScript / Tailwind CSS 4 / vite-plugin-pwa (Workbox) / Fontsource / Vitest + Testing Library

**Spec:** `docs/superpowers/specs/2026-07-20-display-font-settings-design.md`

## Global Constraints

- ブランチは `feat/display-font-settings`（作成済み。spec コミットが載っている）。
- 各タスクのコミット直前に必ず実行する: `mise exec -- pnpm run typecheck && mise exec -- pnpm run lint && mise exec -- pnpm run format:check && mise exec -- pnpm test && git diff --check`（以下「品質チェック一式」）。
- pnpm は直接叩けない環境のため、常に `mise exec -- pnpm ...` の形で実行する。
- コミットメッセージは Conventional Commits 形式の日本語。末尾に `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>` を付ける。
- コードコメントは日本語。非自明な値・外部制約由来の実装には理由コメントを付ける。
- ソースコードの文字列を探索するだけのテストは書かない。
- フォント設定は表示レイヤーのみに作用させる。法令原文・保存データ・コピー内容に影響させない。
- 設定の内部値は次の 6 値で全タスク共通: `"system" | "noto-serif-jp" | "biz-udmincho" | "zen-old-mincho" | "noto-sans-jp" | "biz-udgothic"`。
- 保存キーは `surasura:display:law-font`（本文）と `surasura:display:ui-font`（UI）。
- 表示名は次で統一: システム標準 / Noto 明朝 / BIZ UD 明朝 / ZEN オールド明朝 / Noto ゴシック / BIZ UD ゴシック。

## 事前調査で確定済みの事実

計画時に検証済み。実装者が再調査する必要はない。

- Fontsource パッケージは 5 つとも v5.3.0 で npm に実在する。
- 提供ウェイト: Noto 2 書体は variable（全ウェイト）。BIZ UDMincho / BIZ UDGothic は 400・700 のみ。Zen Old Mincho は 400・500・600・700・900。
- Fontsource の font-family 名: `"Noto Serif JP Variable"` / `"Noto Sans JP Variable"` / `"BIZ UDMincho"` / `"BIZ UDGothic"` / `"Zen Old Mincho"`。
- Workbox の `globPatterns` に woff2 は含まれないため、フォントがプリキャッシュされる心配はない。
- `src/app/manifest.test.ts` は workbox オプションを参照しないため、PWA 設定の変更で壊れない。
- 条文カード・復習画面の本文は現在 `font-serif` を使っておらず、ゴシック（`font-sans` 継承）表示である（Task 0 の spec 修正の根拠）。
- Tailwind 4 の `@theme inline` は utility に変数の「値」を inline 展開する。実行時切替には、色トークンと同様に `@theme` の変数から `:root` の変数を参照させる二段構えが必要（Task 2 のパターンの根拠）。

### Task 7: Workbox にフォントのオンデマンドキャッシュを追加

**Files:**

- Modify: `src/core/pwa/config.ts`

**Interfaces:**

- Produces: service worker の CacheFirst キャッシュ（cacheName `fonts`）。

- [ ] **Step 1: runtimeCaching を追加**

`src/core/pwa/config.ts` の `workbox` オブジェクトへ、`globIgnores` の後に追加する:

```ts
    // 同梱 Web フォントはプリキャッシュせず、描画で使ったチャンクだけを取得後にキャッシュする。
    // 全書体・全サブセットのプリキャッシュは起動時のキャッシュ容量を圧迫するため、
    // OCR 資産（globIgnores）と同じオンデマンド方針を取る。
    runtimeCaching: [
      {
        // Vite が emit する自オリジンのフォントチャンク（/assets/*.woff2）に限定する。
        urlPattern: /\/assets\/[^/]+\.woff2$/,
        handler: "CacheFirst",
        options: {
          cacheName: "fonts",
          expiration: {
            // 日本語フォントは 1 書体が百数十のサブセットに分割されるため、
            // 本文・UI の 2 書体を常用しても収まる数を確保する。
            maxEntries: 512,
            // ファイル名にハッシュが付き内容が変わらないため、1 年の長期保持でよい。
            maxAgeSeconds: 60 * 60 * 24 * 365,
          },
        },
      },
    ],
```
