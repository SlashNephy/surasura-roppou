# LawNode 正規化 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** e-Gov JSON 本文を、ビューア・目次・保存機能が共有できる安定した `LawNode[]` に正規化する。

**Architecture:** e-Gov API の取得と response validation は `src/core/egov/repository.ts` に残し、本文ツリーの正規化は `src/core/egov/lawText.ts` に分離する。`LawNode` は `rawText`、`plainText`、`normalizedText` を持ち、Issue #16 の読みやすさ変換が入るまでは `normalizedText` を `plainText` と同じ値にする。

**Tech Stack:** TypeScript 6, Vitest, e-Gov 法令 API Version 2 JSON law text fixture.
