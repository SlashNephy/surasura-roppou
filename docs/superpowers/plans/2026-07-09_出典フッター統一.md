# 出典明記の共通フッター 実装計画

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 全画面共通の `contentinfo` フッターで e-Gov 出典と免責を必ず表示し、既存のバラバラな出典表記を一本化する。

**Architecture:** 全ルートを包む唯一のレイアウト `AppShell` にインライン `Footer` を追加する（`Header` / `MobileNavigation` と同じ配置様式）。ルートを flex-column 化してスティッキーフッターにし、モバイル固定ナビとの重なりは下部余白で回避する。追加後、法令ビューアと設定画面の重複出典表記を削除する。

**Tech Stack:** React 19 / TanStack Router / Tailwind CSS 4 / Vitest + Testing Library。

## Global Constraints

- 出典行の文言とリンク: テキストは `出典：` に続けて `e-Gov 法令検索（https://laws.e-gov.go.jp）`。この `e-Gov 法令検索（https://laws.e-gov.go.jp）` 全体を `<a href="https://laws.e-gov.go.jp" target="_blank" rel="noreferrer">` にする。「出典：」の区切りは全角コロン `：`。
- 免責文（1 字一句このまま）: `本アプリは学習補助であり、法的助言を提供するものではありません`。
- ESLint は strict type-checked（`pnpm lint`）。未使用 import・未使用関数を残さない。
- コメントは日本語。コミットは Conventional Commits。`Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>` を付与する。
- 参照: 設計 `docs/superpowers/specs/2026-07-09-footer-attribution-design.md`。
