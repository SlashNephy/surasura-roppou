# 構造見出しと目次の数字変換 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 「読みやすい表示」で編、章、節、款、目の番号を本文とデスクトップおよびモバイル目次で算用数字へ変換し、「原文表示」で原文へ戻す。

**Architecture:** 原文の `LawNode` と `LawTocItem` は変更せず、法令構造単位は見出し専用の `transformReadableHeadingText` で描画時だけ変換する。汎用本文変換は一般文章中の章参照を保持する。目次コンポーネントは `displayMode` を受け取り、本文見出しと同じ見出し専用変換を表示タイトルへ適用する。法令ビューワー画面は一つの表示モードを本文と二つの目次へ渡す。

**Tech Stack:** React 19、TypeScript 6、Vitest、Testing Library、playwright-cli、mise、pnpm

---

- [x] **Step 3: 法令構造単位を変換する最小実装を書く**

`readability.ts` では既存の条項号用変換を維持し、文字列先頭の完全な構造番号だけを認識する `transformReadableHeadingText` を追加する。
構造番号は編、章、節、款、目と任意の枝番号からなり、その直後が終端、空白、句読点、記号の場合だけ変換する。

汎用 `transformReadableText` には構造番号変換を追加しない。
本文見出しのタイトルと対応する本文接頭辞、目次タイトルからだけ見出し専用変換を呼び出す。

不正な基本番号は枝番号を含む構造接頭辞全体を保持し、後続の独立した条番号や別表には既存の汎用変換を適用する。
