## 2. コード品質のフォローアップ（最終レビュー指摘）

- [ ] **ルート union の共有化**。`src/app/SearchPalette.tsx` の `PaletteDestination["to"]` と `src/app/AppShell.tsx` の `NavItem["to"]` で `"/laws" | "/scanner" | "/study" | "/settings"` が重複している。ルート追加時に 2 箇所の修正漏れが起きるため、union 型だけ共有モジュールへ切り出す。ラベルの差（「法令」と「法令を探す」）は意図的なので、定義全体の統合はしない。
- [ ] **日付整形ヘルパーの共通化**。同一ロジック（`slice(0, 10)`、失敗時「不明」）が 3 箇所にある: `src/core/viewer/LawDocumentView.tsx` の `getDisplayFetchedDate`、`src/app/pages.tsx` の `formatSavedLawFetchedDate`、`src/app/law-viewer-page.tsx` の `getDisplaySourceDate`。共通 util へ抽出する。抽出すれば `getDisplaySourceDate` の「不明」分岐（現状唯一未テスト）が既存テストで担保される。
- [ ] **`useSavedLaws` フックの抽出**。`src/app/home-page.tsx` と `src/app/pages.tsx`（LawsPage）で、`useEffect` + `isCurrent` ガード + エラー state の保存済み法令ロードパターンが丸ごと重複している。
